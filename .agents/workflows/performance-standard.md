# パフォーマンス維持のための開発ルール (AIエージェント用)

このドキュメントは、AIエージェントが当サイト（ドラえもんF同好会）を修正・機能追加する際に遵守すべきパフォーマンス基準を定義したものです。PageSpeedスコア100点を維持するために、以下のルールを必ず守ってください。

## 1. アセット（画像・フォント）の管理
- **画像配置**: 新しい画像は `src/assets/images/` 配下に置いてください。`public/` への直接配置は、ハッシュ化と最適化の恩恵を受けられないため避けてください。
- **フォント**: 外部の Google Fonts へのリンクを追加しないでください。新しいフォントが必要な場合は、WOFF2形式を `src/assets/fonts/` にダウンロードし、`global.css` で `@font-face` を定義してください。
- **プリロード**: ブラウザが早期に発見すべき重要なアセット（フォント等）は、`BaseHead.astro` で `<link rel="preload">` を設定してください。

## 2. アニメーションの実装
- **コンポジット重視**: アニメーションは必ず `transform` (scale, translate, rotate) および `opacity` のみを操作してください。
- **禁止事項**: `box-shadow`, `width`, `height`, `top/left`, `filter` など、レイアウトやペイントを誘発するプロパティをアニメーションさせないでください。
- **オーラ効果**: 光るエフェクト（オーラ）を作成する場合は、`box-shadow` を動かすのではなく、擬似要素（`::after` 等）で `scale` と `opacity` を組み合わせて表現してください。

## 3. スクリプトの最適化
- **コンポーネント管理**: JavaScript ロジックは `src/components/` 内の各 `.astro` ファイル内の `<script>` タグに記述してください。グローバルな巨大 JS ファイルの作成は避けてください。
- **強制リフローの回避**: スクロールイベントやサイズ変更イベントで DOM を操作する場合は、必ず `window.requestAnimationFrame()` を使用し、読み取り（`window.scrollY` 等）と書き込み（`classList.add` 等）を適切に分離してください。
- **パッシブ・リスナー**: スクロールイベントには `{ passive: true }` を設定してください。

## 4. ビルドと検証
- 修正完了後は必ず `npm run build` を実行し、ビルドエラーが発生しないことを確認してください。
- 可能であれば、ビルド結果の HTML を確認し、アセットのパスが正しくハッシュ化されているかチェックしてください。
